# 프로젝트: 영어 단어 시험 생성 및 채점 웹 서비스 (MVP)

## 목표

영어 단어 시험을 위해:

1. 단어 시험지 자동 생성 (오지선다)
2. PDF 출력
3. 교사 입력 기반 자동 채점
4. 결과 저장

---

# 핵심 설계 원칙

* MVP 기준으로 **가장 단순하고 빠르게 구현**
* 자동 채점은 **교사 입력 방식 사용**
* DB는 **Google Sheets, JSON, 무료DB**
* 웹 기반 (Next.js 추천)

---

# 기능 1: 시험 문제 생성

## 입력

* DAY 선택 (예: DAY1)
* 문제 수 (옵션: 전체 or N개)

## 로직

1. 해당 DAY의 단어 리스트 조회
2. 각 단어에 대해:

   * sentence에서 해당 word를 찾아 빈칸 처리
   * 예: "thirst" → "_____"
3. 보기 생성:

   * 정답 1개 + 랜덤 오답 4개
   * 오답은 전체 DB에서 랜덤 선택
4. 보기 순서 랜덤 섞기

## 출력 구조

```json
{
  "question": "The drink would quench the local population’s _____.",
  "choices": ["thirst", "investment", "precision", "ambiguity", "phenomenon"],
  "answer": 1
}
```

# 기능 2: PDF 시험지 생성

## 요구사항

* 문제 번호 포함
* 객관식 5지선다
* 학생 이름 작성 공간
* 깔끔한 인쇄용 레이아웃

## 구현 방법

* HTML 템플릿 생성
* Puppeteer로 PDF 변환

---

# 기능 3: 교사 입력 채점 시스템

## UI 요구사항

### 화면 구성

* 학생 선택 드롭다운
* 문제별 선택 버튼

예시:

```
학생: [홍길동 ▼]

1번  [①] [②] [③] [④] [⑤]
2번  [①] [②] [③] [④] [⑤]
...
```

---

## 동작 방식

1. 교사가 학생 시험지 보고 클릭 입력
2. 제출 버튼 클릭

---

## 채점 로직

```js
score = 맞은 개수
wrong_words = 틀린 문제의 word 리스트
```
