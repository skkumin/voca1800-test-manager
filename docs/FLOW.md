└──────────┬──────────┘
           │
    ┌──────▼──────┐
    │ DAY 선택?   │
    └──┬───────┬──┘
       │       │
   YES │       │ NO
       │       │
       │  ┌────▼────────────┐
       │  │ 미출제 예문만   │
       │  │ 출제 진행       │

       │       │
    ┌──▼───────▼──┐
    │  DAY1,2,3   │

    ┌──────▼────────────────┐

    └──────┬────────────────┘
           │
    ┌──────▼──────────────────┐
    │ 시험 미리보기 페이지    │

## 상태 전이

| 페이지 | 입력값 | 다음 페이지 | 비고 |
|--------|--------|-----------|------|
| / (홈) | DAY + 문제수 | /test/[testId] | POST 생성 후 redirect |
| /test/[testId] | - | (PDF DL) | 미리보기 전용 |
| /grade | test_id + 답 | (결과 표시) | POST 채점 후 inline |
| /results | - | - | 조회 전용 |

---

## 핵심 API 흐름

```
홈 화면
  ↓
POST /api/test/generate
  ├─ Mode A (DAY): WHERE day IN ['DAY1', 'DAY2']
  ├─ Mode B (미출제): sentence_usage 이용 조회
  ├─ 예문 출제 이력 확인
  ├─ N개 단어 + 예문 선택
  ├─ tests 테이블 저장
  └─ test_id 반환
  ↓
/test/[testId]
  ├─ GET /api/test/[testId] (문제 조회)
  ├─ React-PDF 렌더링
  ├─ PDFDownloadLink
  └─ PDF 다운로드
  ↓
/grade
  └─ POST /api/grade
      ├─ tests.questions에서 정답 조회
      ├─ 제출한 answers와 비교
      ├─ score + wrong_words 계산
      ├─ test_results 저장
      ├─ sentence_usage 기록 (출제 이력)
      └─ 결과 반환
```

## 미출제 예문 추적 흐름 (핵심)

```
출제 前:
  words.sentences = ['예문1', '예문2', '예문3']
  sentence_usage = (empty)

출제 A (DAY1):
  1. sentence_index=0 (예문1) 선택 → 출제
  2. INSERT sentence_usage (word_id, 0, test_id_A)

출제 B (DAY1 재출제):
  1. SELECT * FROM sentence_usage WHERE word_id=X
     → index=0 (used_count=1)
  2. index=1,2는 used_count=0 → 우선 선택
  3. index=1 (예문2) 선택 → 출제
  4. INSERT sentence_usage (word_id, 1, test_id_B)

결과:
  같은 DAY 재출제도 예문이 중복되지 않음 ✓
```
